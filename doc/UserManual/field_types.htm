Field Types

 Top  Previous  Next

Field types define how listing information and data of users profiles are entered, stored and displayed.

 

1.Boolean - has only one of two values: true or false (yes or no). You can use it for the data as Air Conditioning, Anti-lock brakes, Power Mirrors and etc.
2.Calendar - stores the calendar data (reservation periods) for a listing. Please refer to the "Availability Calendar" article to learn more.
3.Decimal - represents the numbers which are too large or too small to be represented as integers (0.26; 4.36; 5689.03 etc.). It should be used for the fields which require the storage of exact values; however, for money values like Price or Rent use the type "Money". You must specify the minimum and maximum values and a number of digits after the decimal separator for the fields of this type in the settings of a field of this type.
4.File - keeps any type of uploaded files. Visitors can download these files. You can specify the maximum size of the files for upload in the settings of a field of this type. By default, 'Maximum File Size' = '0'. If the maximum size of the uploaded file is not specified, there is no limitations set by iLister system. Please note that the file name should contain only Latin letters, otherwise the file name will be changed after the upload.
5.Float - represents the numbers which are too large or too small to be represented as integers (0.23; 5.869; 3209.01 etc.) It can be used for fields like Mileage, Lot Size, Apartment Square etc., which do not require the exact storage of the values entered by a user in contrast to decimal-type fields. You must specify the minimum and maximum values and a number of digits after the decimal separator for the fields of this type in the settings of a field of this type.
6.Known Geographic Locations - contains the information an geographic locations and their coordinates: longitude and latitude which allow visitors to search within particular radius from his location.
7.Integer - natural numbers including 0 (0, 1, 2, 3, ...) together with the negatives of the non-zero natural numbers (−1, −2, −3, ...). It is used for numerical values such as Year, Number of Doors and etc. You must specify the minimum and maximum values for the fields of this type in the settings of a field of this type. It gives an opportunity to make a search according to exact match, certain range, more then entered value, less then entered value, above or equal entered value, less or equal entered value.
8.List - a set of values predetermined by administrator. Users/visitors cannot add their own options there.
9.Multi-List - a set of Boolean values predetermined by administrator. For example, Vehicle Options field with type Multi-List may contain Air Conditioning, Power Steering and etc.
Site administrator can add multiple values to the list-type fields at once using the "Import List Values" feature in Admin Panel -> Edit list-type field -> Import List Data. You can import values from CSV or XLS formats. It is necessary to specify a column name in the importing file; the column name is not important, you can put there anything as long as the first line with the column name is ignored during the field value import. Please note that for the XLS format you can import .xls files of the following versions only: MS Office'95, MS Office'97, MS Office'2000, MS Office'2003; and you cannot import .xlsx files.
10.Money - represents decimal numbers and should be used for the fields which stores all type of prices like Price or Rent. You must specify the minimum and maximum values and a number of digits after the decimal separator for the fields of this type in the settings of a field of this type.
11.Rating - gives the ability for users to vote and rate listings. The rating is calculated as an average from all votes.
12.String - a short data which can contain alpha-numeric and special symbols. The administrator can restrict the string values length through the administration panel. By default, the maximum length is 256 symbols. You can use string for fields like City, Address, VIN etc.
13.Text - a large data with text; this type is commonly used for storing arbitrary text values such as Description, Seller Comments etc.
14.Tree - data that emulates a hierarchical list structure with a set of linked nodes. A good example is State and Province lists. Every State has a particular set of  Provinces, so we can say that Provinces list is hierarchically subordinated to States list.
15.Video - uploads video-type files to YouTube. Visitors can view them on the Listing Details page. You can specify the maximum size of the files for upload in the settings of a field of this type. By default, 'Maximum File Size' = '0'. If the maximum size of the uploaded file is not specified, there is no limitations set by iLister system. Please note that the video file name should contain only Latin letters, otherwise the file name will be changed after the upload. Supported video formats are: 3gp, asf, asx, avi, mov, mp4, mpg, qt, rm, swf, wmv . In order to be able to upload video, website Administrator needs to make appropriate configurations. Also you need to make sure that php post_max_size and upload_max_filesize are enough to upload the video. These values should be more than 'Maximum File Size'.
