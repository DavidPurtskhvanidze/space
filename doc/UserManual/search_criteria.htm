Search Criteria

 Top  Previous  Next

iLister uses a unified criteria sets to pass search criteria from search forms to the search results page. Information given in this section of the manual will help you to understand how unified criteria can be used to build search forms and pass criteria to the search functionality of iLister, which should result in page displaying relevant search results (listings or users). The unified criteria can be applied to both listings and users. After reading this section you should be able to create search forms and links to search results pages using iLister as well as using static HTML or 3rd party software.

 

How Criteria Are Formed

Criterion, applied to the searchable object (listing or user) is comprised of the following parts:

1.id of the field the criterion applies to (can be either listing or user profile field)
2.type of the criterion
3.value or key-value pairs associated with the criterion (allowed set of keys are defined by a criterion type)

 

These parts are combined in the following way:

1.first comes the field id
2.next, the criterion type is concatenated in square brackets
3.then, if it is a single-value criterion there is a equals sign "=" followed by value or
4.if it is a multi-key criterion, criterion type is followed by the key in square brackets, followed by "=" and value pertaining to the key

 

Here is an example of single-value criterion: City[equal]=Boston. In this example City is the id of the field, equal is the type of criterion and Boston is the value. This example demands that the object's field City equals to "Boston".

 

Here is an example of multi-value criterion: Zip[geo][radius]=100&Zip[geo][location]=10001. This criterion set demands that ZIP code (stored in the Zip field of the object) of the object being search should be within 100 miles (or kilometers) from the ZIP code 10001.

 

Here are more examples:

•category[equal]=HomesForSale
•Zip[geo][radius]=any
•Zip[geo][location]=90001
•Price[not_less]=500
•Price[not_more]=2000
•keywords[includes_all]=park+lake+playground

 

All known criterion type and their keys are listed in the Criterion Type Reference section of the manual.

 

How Search Criteria Are Passed From Forms To Search Results

The search criteria are passed to search results pages in a form of HTTP GET or POST parameters, either as a part of the query string or part of the the submitted form data. Thus, we can include the criteria in the URL or create them using HTML forms.

 

Here is an example of the URL of the listings search results page with a bunch of criteria:

http://demo.worksforweb.com/iRealty/search-results/?City[equal]=New+York&State[equal]=&YearBuilt[not_less]=1995&YearBuilt[not_more]=2011&Basement[equal]=1&action=search

 

Please note the following regarding the URL above:

1.There is an action=search parameter in query string. It triggers the search. Thus, it must be included in the URL. (Though not necessarily needs to be the fist parameter)
2.The value of the city[equal] criterion has a "+" instead of the space. This is because the query sting should be URL encoded.
3.There are empty parameters in the query string (State). These are usually a result of the submitted form had some of its fields left blank. The blank (and invalid) criteria are ignored.

 

How to Use Search Criteria

•You can pass search criteria in a query string which is a part of a URL that contains data to be passed to web applications like iLister.

 

For example, you may want to put a link on your site which leads to a page with all Homes For Rent in New York. It will look as follows:

<a href="{$GLOBALS.site_url}/search_results/?category[equal]=HomesForRent&action=search&City[equal]=New+York">Rentals in New York</a>

Please note that the format of the URL is URL encoded.

 

 

•Another example is how to create a search form on another (not iLister-based website). If you sell cars and represent a dealership, you can have a search form on the dealership site which leads to the specific search results on your iLister-based site. The code of the form will be like the following:

<form action="www.yoursite.com/search_results/">

<input type="hidden" name="action" value="search">

<input type="hidden" name="State[equal]" value="CA" />

[[Find listings in]] <input type="text" name="City[equal]" value="" />, CA

<input type="submit" class="button" value="Find"/>

</form>

 

In this case a search form on the dealership site will lead to your site at http://www.yoursite.com/search_results/ and will pass the search criteria specified in HTML <input> tags to the query string. In this particular case, the criteria are: the state which is set to "CA" in hidden field and the city, which site user are able to type into the text box before clicking the the "Find" button.

 

•And finally you can create a search form on your iLister-based website using its built-in tools:
1.create a template for the search form (you can use default_search.tpl as an example) and specify desired search parameters and criteria there:

<form method="GET" action="{$GLOBALS.site_url}/search_results/">

<input type="hidden" name="category[equal]" value="{$category_id}" />

<input type="hidden" name="action" value="search">

 <table border="0" cellpadding="2" cellspacing="0">

         <tr>

                 <td width="45">[[$form_fields.City.caption]]:</td><td>{search property=City}</td>

                 <td align="right" width="45">[[$form_fields.State.caption]]:</td><td>{search property=State}</td>

         </tr>

         <tr>

                 <td width="45">[[$form_fields.Price.caption]]&nbsp;{$GLOBALS.custom_settings.listing_currency}:</td>

                 <td colspan="3">{search property=Price}</td>

         </tr>

         <tr>

                 <td colspan="2" ><a style="font-weight:normal;" href="{$GLOBALS.site_url}/search/">[[Advanced search]]</a></td>

                 <td align="right" colspan="2">

                         <input type="submit" class="button" value="[[Find:raw]]"/>

                 </td>

         </tr>

 </table>

</form>

 

In iLister search form you can use the construction {search property=$fieldID} to specify the field to search by. Depending on the type of the field iLister will replace the {search ...} tag with an HTML form elements with the help of search template specified for this field type. For example, the search template for the field type "list" is list.tpl and search criteria is [equal]:

<select name='{$id}[equal]' class="searchList">

 

You can override the default search template for the field using the "template" parameter to the {search ...} tag, i.e. {search property=State template=my_list.tpl}. Please mind that template specified in the "template" parameter should be placed into the /modules/field_types/apps/FrontEnd/templates/$ThemeName/search/ folder.
