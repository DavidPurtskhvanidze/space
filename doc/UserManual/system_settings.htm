System Settings

 Top  Previous  Next

System settings are values that used in the system to set up the system work.

 

General Settings

•Listing Picture Width — the width of pictures which users will upload for their listings (pixels).
•Listing Picture Height — the height of pictures which users will upload for their listings (pixels).
•Listing Thumbnail Width — the width of thumbnails created from pictures which users will upload for their listings (pixels).
•Listing Thumbnail Height — the height of thumbnails created from pictures which users will upload for their listings (pixels).
•Listing Large Picture Width — the width of big pictures created from pictures which users will upload for their listings (pixels).
•Listing Large Picture Height — the height of big pictures created from pictures which users will upload for their listings (pixels).
•JPEG Image Quality — the quality of jpeg images for listings and user profiles. The highest is 10 (for those who wants to have high-quality images on the site), the lowest is 0 (if someone wants to keep disk space).
•Listing Picture Storage Method — pictures can be stored in the File System or in the Database.
•Watermark image, opacity and position — the settings for the image that is applied as watermark to all listing pictures when uploaded to the website.
•Favicon icon — icon applied as favicon to the website.
•User Input Escaping — sets the way how the information entered by users and visitors is displayed.
oNo escaping (unsafe, XSS-vulnerable) — no HTML tags escaping (Caution! This setting value is opening an XSS  threat: http://en.wikipedia.org/wiki/Cross-site_scripting)
oEscape all HTML tags — full HTML tags escaping; special symbols <,>,",& will be changed to &lt;, &gt;, &quot;, &amp;. This option is not compatible with WYSIWYG editors because all rich content (in HTML) created using WYSIWYG editor will be escaped. Make sense if you WYSIWYG editor is disabled.
oRemove potentially unsafe tags with HTML purifier — use HTML Purifier v.4.4.0 Standalone (http://htmlpurifier.org/) library to strip potentially harmful tags from the input before displaying it on the website. It is a recommended option, because it is both compatible with HTML generated by WYSIWYG editors and provides sensible security.
•Google Maps API key — Google Maps API key Configuration
•Enable WYSIWYG Editor — sets whether WYSIWYG editor or a basic text area is displayed for listing description (all "text" typed listing fields) and other form of the iLister-based website. Please a corresponding feature description further in the manual
•Enable Share Block — specifies if the sharing option is enabled and displayed on your website; so if you like, you can turn it off and save some traffic between your website and AddThis.
•Enable autocomplete for keyword search — the website administrator can turn on/off the autocomplete for keyword search.
•Response for deleted and inactive listings — lets the administrator to choose the HTTP response for deleted and disabled ads.

Language and Regional Settings

•Default Domain — the default domain for search phrase translation. If iLister cannot identify which of domains the found phrase belongs to, it will expect the domain specified in this setting.
•Default Language — the default language for site. Settings for this language are used for the date format in Admin Panel.
•Mark Phrases That Are Not Translated — makes iLister treating phases that do not have a translation to a current language:
odefault — display the translation of a phrase for the current language or the ID of a phrase in case the translation is missing. You can use this setting on a live website.
ohighlight — put a "(T)" link near the phrase, which links to the translation interface. This setting is very useful for revealing missing translations which you can add to the language files. However, this configuration should not be used as a permanent setting on live sites.
•Radius Search Unit — either Miles or Kilometers. Sets the units that are considered by iLister when performing the ZIP/radius search.

 

 

 

Email and Notifications

•System Email — the website uses this email address to send notification emails to users and website administrator.
•Notification Email — the email address where the notification email will be sent.
•Reply-To — email used as a reply-to address when recipients reply to a newsletter.
•Return-Path — email address to which bounce messages are delivered.
•Notify when User Added a Listing — this flag means that the notification about listing adding will be sent.
•Notify on Comment/Reply Added —  makes iLister send email notification to site administrator when users post Comment or Comment Reply.
•Notify when Listing has Expired — this flag means that the notification about listing expiration will be sent.
•Notify when User Contract has Expired — this flag means that the notification about user contract expiration will be sent.
•Notify when User has Registered — this flag means that the notification about a new user registration will be sent.
•Notify user when balance is lower the threshold — available with Credit System module only. Enables iLister to send notification to users, who's credit balance is below a given threshold. Notification message warns user if his/her balance being low and prompts to purchase more credits.
•Balance notification threshold — available with Credit System module only. Sets the threshold for the low balance notification.
•Number of days prior to listing or subscription expiration to send out notification to users — the ability to specify the number of days between listing or subscription expiration and the date of related notifications sent to a user.

QR Code

•Error correction capacity — QR Code has error correction capability. Data can be restored even if the symbol is partially dirty or damaged. A maximum 30% of codewords can be restored (Data restoration may not be fully performed depending on the amount of dirt or damage):
1.Level L        Approx. 7% of codewords can be restored.
2.Level M        Approx. 15% of codewords can be restored.
3.Level Q        Approx. 25% of codewords can be restored.
4.Level H        Approx. 30% of codewords can be restored.
•Code square size — the size of the squares units in pixels. For example, the number 4 means that the square has sides equal to 4 pixels each.
•Boundary size — the size of margins which surround a QR code in order to make it isolated on the printing or display area. The isolation makes the QR code recognition  easier for mobile devices.

For additional information please refer to the page "QR Code Generator" and its settings.
