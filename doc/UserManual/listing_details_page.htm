Listing Details Page

 Top  Previous  Next

 

The template which displays Listing Details information is default_view.tpl which can be found in

Admin Panel -> Site Layout -> Module Templates -> Classifieds Engine -> category_templates/display/default_view.tpl

 

The following sections can be modified in default_view.tpl:

•Manage Search
•Rating
•Seller Info

 

Rating, Facebook and Twitter

Rating, FB and Twitter are displayed by the code

{include file="category_templates/display/social_network_buttons.tpl"}

<span class="fieldValue fieldValueListingRating">{display property=ListingRating}</span>

 

The template which is used for rating-type fields is rating.tpl in Admin Panel -> Site Layout -> Module Templates -> Classifieds Engine. There you can see the images that are used for rating stars, and you can easily change them.

 

 

Seller Information

The seller block starts with the code shown below:

<div class="head"><span>[[Seller Info]]</span></div>

<div class="manageSearchOptions">

 {include file="author_info.tpl" listing=$listing}

</div>

 

You can add more information about the seller to this section from User Profile fields of the User Group.

 

 

Listing Caption

Listing caption is built from the "Listing Caption" category setting and the listing fields specified in the following code:

<div class="listingDisplayHeader">

 <h1>

         {if $listing.Sold.exists && $listing.Sold.isTrue}

                 <span class="fieldValue fieldValueSold">[[SOLD]]!</span>

         {/if}

         {$listing} 

         {if $listing.Price.exists && !$listing.Price.isEmpty}

                 <span class="fieldValue fieldValuePrice {$listing.Price.type}">{$GLOBALS.custom_settings.listing_currency}[[$listing.Price]]</span>

         {/if}

 </h1>

</div>

 

You can specify any listing field to compose a caption which helps you to create a SEO-friendly title for this page.

 

 

Listing Tabs

The tabs divide listing information on several sections that is more comfortable to view.

You can modify tabs in the class "listingDetailTabs tabularPages".

 

 

How to Add a New Field

You may want to add a new field to the "Summary" tab, and to do so, you need to edit the file:

\modules\ilister_tabs_blue\classifieds\category_templates\display\listing_view_summary.tpl

 

By default, iLister displays all fields of the listing category in the foreach cycle:

{foreach from=$form_fields item=form_field}

 

Thus, when you create a new field, it is automatically added to the Listing Details template.

 

However, you may want to add a new field which will be displayed in a different way. You can do this as follows:

{assign var="fieldsToExclude"

 value=[

         'sid',

         'id',

         'active',

         'pictures',

         'NewField',         //Exluding your new field from the foreach cycle

         ...

]}

 

You need to change "NewField" to the ID of your new listing field.

 

 

Listing Controls

The listing controls block represents the visitor and/or user actions related to the listing. This section is included to listing_view_summary.tpl in the file \modules\classifieds\apps\FrontEnd\templates\category_templates\display\listing_details_listing_controls.tpl:

<td class="listingControlAndAdditionalInfoWrapper">

 {include file="category_templates/display/listing_details_listing_controls.tpl"}

 <div class="additionalInfo">

         <span class="fieldCaption fieldCaptionActivationDate">[[FormFieldCaptions!Date Posted]]:</span> <span class="fieldValue fieldValueActivationDate">[[$listing.activation_date]]</span><br />

         <span class="fieldValue fieldValueViews">{$listing.views} [[views]]</span>

 </div>

</td>

 

In order to add more controls links or change/remove some of them you need to go to

Admin Panel -> Site Layout -> Module Templates -> Classifieds Engine -> category_templates/display/listing_details_listing_controls.tpl

<li>

 <a href="{$GLOBALS.site_url}/print-listing/?listing_id={$listing.id}" onclick="return openLinkInWindow(this,'_blank');"><img class="linkIcon" src="{url file='main^icons/print.png'}" alt="&#8226;" /></a>

 <a href="{$GLOBALS.site_url}/print-listing/?listing_id={$listing.id}" onclick="return openLinkInWindow(this,'_blank');">[[Print This Ad]]</a>

</li>

 

 

 

Listing Images

Listing images are displayed under the tab "Pictures":

{if $listing.pictures.numberOfItems > 0}

 <li class="tab pictures"><a href="#Pictures">[[Pictures]]</a></li>

{/if}

 

In order to change the section with listing images, you need to modify the file listing_images.tpl in Admin Panel -> Site Layout -> Module Templates -> Classifieds Engine. The CSS styles that define the look of the image section, are specified in the listing_images.tpl file. It includes the file for displaying image slideshow, if this option is enabled for the listing:

{module name="listing_feature_slideshow" function="display_slideshow" listing=$listing}

 

 

Google Map Details

The map locates under the "On the Map" tab:

<li class="tab onTheMap"><a href="#OnTheMap" id="googleMapDisplayControl">[[On the Map]]</a></li>

 

The listing details page connects to the map template \modules\google_map\apps\FrontEnd\templates\google_map.tpl via the code:

<div id="OnTheMap" class="tabContent locationOnMap">

 <div class="fieldValue fieldValueLocationOnMap">

 {module name='google_map' function='display_map' address=$listing.Address|cat:", "|cat:$listing.City|cat:", "|cat:$listing.State default_latitude=$listing.ZipCode.latitude default_longitude=$listing.ZipCode.longitude}

</div>

 

The Google Map details are retrieved from the following data:

{getCoordsByAddress address=$listing.Address|cat:", "|cat:$listing.City|cat:", "|cat:$listing.State assign=geoData}

{if empty($geoData) && !empty($listing.ZipCode)}{assign var=geoData value=$listing.ZipCode}{/if}

{if !empty($geoData)}

{include file="category_templates/display/listing_details_googlemap.tpl" geoData=$geoData}

{/if}

 

Here is what is happening:

1.{getCoordsByAddress address=$listing.Address|cat:", "|cat:$listing.City|cat:", "|cat:$listing.State assign=geoData}

iLister creates the variable geoData with the data from listing fields Address, City and State. You can change this set of fields to your own.

2.{if empty($geoData) && !empty($listing.ZipCode)}{assign var=geoData value=$listing.ZipCode}{/if}

In case geoData is empty (i.e. Address, City and State were not filled for this listing) and there is a ZipCode for this listing, then the ZipCode value is passed to geoData.

3.{if !empty($geoData)}

{include file="category_templates/display/listing_details_googlemap.tpl" geoData=$geoData}

{/if}

Then, if there is any information in geoData (either from Address, City and State or from the ZipCode), iLister displays the block with the map which shows the location based on the data from geoData.

 

 

Comments Section

Listing comments are under the "Comments" section:

<div id="Comments" class="tabContent comments">

 {assign var='current_uri' value=$GLOBALS.current_page_uri|cat:'?action=restore&'|cat:$smarty.server.QUERY_STRING}

 {assign var='current_uri' value=$current_uri|urlencode}

 {module name="listing_comments" function="display_listing_details_comments" listing=$listing returnBackUri=$current_uri}

</div>

 

Comments section is linked to the listing details page through listing_details_comments.tpl that can be found in

Admin Panel -> Site Layout -> Module Templates -> Classifieds Engine -> category_templates/display/listing_details_comments.tpl

 

The CSS style for the comments section is <div class="comments">.
