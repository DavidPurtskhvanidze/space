Basic concepts: Domains and Phrases

Top  Previous  Next

In order to translate your site you need to translate phrases. Phrases belong to domains.

 

Each language file contains domains and phrases which represent the actual translation which is visible on the front-end.

 

Domain is a category of phrases, which determines their context.

Phrase is the ID of the text which you want to translate.

 

Domains

When you make a translation you can add it to any domain you like, either existing or a new one.

 

However, we strongly recommend for you to add translation for the fields of list- and tree-type fields to their own domains, which must have the mask like Property_FieldID, e.g.:

Property_BodyStyle

Property_MakeModel

Property_State

 

 

 

In this case your iLister-based website will know where (in what domain) it can take the translation for list- and tree-type fields.

 

However, you can make the site to take the translation for a phrase in a different domain.

 

How to add a new domain

You can add new domain from the Admin Panel -> language Management -> Manage Phrases -> Add Phrase:

 

When you click the link "Add new domain", the Admin Panel allows you to type the name for new domain:

 

Please note that you can add new domain only along with a new phrase.

 

Alternatively, you can add new domains directly to XML language files:

1.Open the language file of your default language (you should open it on your server in the folder /languages via FTP client or your hosting Control Panel) and add the following line after the  tag <pages> in order to create a new domain:

<page key="Property_FieldID"/>

2.Then save the langfile on your server.
3.Afterwards you will be able to add the values to the “FieldID” phrase (not the FieldID field!) and translate them.

 

Please refer to the example below on how to add a new phrase to learn more.

How to add a new phrase

For example, you want to have a new field “Car Condition”.

1.Create a new list-type field in Admin Panel ->Listing Management -> Categories -> edit “Car” -> click the link "Edit 'Car' Category Fields" -> Add a New Field for the 'Car' Category -> fill in the “Add a New Category Field” form:

Click “Add”.

 

2.Edit a newly created “CarCondition” field there and add values to a list:
•Excellent
•Good
•Poor
•Spares

 

 

3.Add the "Car Condition" caption in Admin Panel -> Language Management -> Manage Phrases -> click "Add new":

Phrase ID = Car Condition

Domain = FormFieldCaptions

Complete the translation for all languages:

 

 

4.Create a new domain for the CarCondition field.
5.To add and translate the values to the “CarCondition” phrase, go to Admin Panel -> Language Management -> Manage Phrases -> click the link “Add New” - > fill in the “Add a New Phrase” form:

Phrase ID = Excellent

Domain = Property_CarCondition

English = Excellent

Complete the translation for all other languages:

 

 

 

You need to add all values for the “CarCondition” phrase this way.

 

The Inline Edit of Phrases

There is no need to reload the page to edit a translation of one language. You can easily do this on the fly: type a phrase or its part in the "Phrase ID" field and choose a desired language on the Manage Phrases page and click "Search".

 

You can see only one translation of phrases — English. Please note that the translations are underlined. If you roll the mouse over a translation, you'll see the caption "Click to Modify".

 

To edit the translation of all available languages, please click the link "Edit".
