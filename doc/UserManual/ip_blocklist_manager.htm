IP Blocklist Manager

Description

IP Blocklist Manager allows site administrator to add IP or IP range to the blocklist thus the visitors from the listed IPs cannot view normal site.

 

IPs are recorded to the database and the .htaccess file (if it has permissions to write) when user creates or edits his listing, or add a comment or reply to a comment. Database and .htaccess store the IP from the last handling only.

 

In case visitor's IP is in the blocklist, the visitor sees either a 403 Forbidden page — in case blocked IPs are stored in the .htaccess file and in the database, or they see the notice, if visitor's IP is recorded to the database only:

 

 

Site administrator can create a custom 403 page: this custom error page can be configured with a single line in the .htaccess  file as shown below:

ErrorDocument 403 /403.html

   

The custom error document itself (403.html) should be a standard formatted HTML document and must be placed to your server at the directory where root .htaccess file is located.

 

IPs are visible for the site administrator only on the pages of listing/comments displaying and listing editing. These pages have the link "Add this IP to Blocklist" (you should put the mouse on a listing record or comment to see the link), which allows administrator to quickly add this IP to the Blocklist through the pop-up window without re-loading of the current page. After the IP was added to the Blocklist, the administrator will see the confirmation saying "The IP $IP has been added to the IP Blocklist."

 

 

 

 

 

 

 

How to Add IP or IP Range

Administrator can add IP or IP range in the section "IP Blocklist" in Admin Panel following the link "Add a New IP / IP range".

The supported formats of single IP and IP ranges are standard for the Apache server v.2.2 and IPv4 addresses. You can learn more about this at the official Apache page about the 'allow' directive. In case a visitor comes from an IPv6 address, iLister will let him enter the website in any case.

 

 

Administrator cannot block his current IP. He will see the warning notice if he tried to block himself.

 

 

Import and Export of IP Blocklist

There is a very useful function of importing IP list from another source to your website in CSV or Excel format and the ability to export the list of IPs.

 

Import works pretty much like the import of users or listings.

You can omit the columns 'added' and 'comment'.

 

As the result of export, administrator gets Excel file with data which were exported:

•IP — single IP or IP range as they were added by administrator;
•Date added — the date when IP range was added;
•Comment — comment for the IP record.

 

 

IP Search Form

Administrator can easily find necessary IP blocking record to edit it or remove from the IP Blocklist. The page with search results of IPs:

•displays total number of IP records;
•provides bulk deletion of IP records;
•represents search results page--by-page, facilitating viewing of records and reducing page loading.
