General Information

 

The user search feature allows your visitor to search for a particular user or user group. User search offers the following functionality:

•Administrator can set up user search pages, configure which user group (or all of them) can be searched and which user profile fields (common or specific) are available for search;
•Visitors are able to search within user groups as set up by the administrator;
•Search results are a list or users found with brief descriptions (additional details can be added by the administrator via templates)
•Each search result item provides links to the complete user profile, all listing by this user and a contact user page (can be modified via templates)

 

Administrator can set up as many user search pages as desired. In order to create a user search page, create a new Site Page with a"search_users” function of the “users” module.

 

When setting up a user search page, please keep the following in mind:

1.The list of fields in the “fields” parameter is used inside the default search form template (user_search_form.tpl). If you use a different template for user search form (by setting a value to the search_form_template parameter) the list in “fields” parameter may be disregarded by the creator of the search form template. Creator of the search template has the choice either to disregard the list from the “fields” parameter or use it.
2.Only those fields that are available in the profile of the user group “user_group_id” make sense as a search criteria. For example if site visitor is looking for the company named “Alpha” in user search, but the group within which search is done does not have the “company name” field no users will be found (empty result). Please keep this consideration in mind when setting up user search page and creating a custom search form template.
3.Administrator can leave the parameter "user_group_id" blank and specify only common user profile fields (like First Name or State). In this case the user search will be performed among all User Groups by specified fields of the user search form. Specific user profile fields will be ignored if the parameter "user_group_id" is blank.

 

The layout of the search results page is configured via the search results template (by default called “user_search_results.tpl” or custom template(s) that contain the result_template parameter).  By editing this template, the administrator sets user profile fields to be displayed and configures field positions and layouts. Please refer to the default template for examples of the user search results layout.

 

Site Visitor Experience

 

When a visitor opens the search user page, he/she will see a search form. Visitor enters search criteria to the form fields and clicks the “Search” button. Following that, search results are displayed below. If no search criteria are entered, all users will appear in the search.

 

Search result page contains brief information about the users found (pretty much like listing search results page).

 

User search results are paginated, with controls to flip pages and set a total number of results per page located above and underneath the results block. To switch pages, click on the page number or on the “Next” or “Previous” links on the page. To change a number or results per page, select the desired number in the drop-down box labeled “Number of records per page”.

 

By default, there are three links below each result:

•“View All Listings” — links to a page where all listings of the user are displayed,
•“Contact User” — links to a page with contact from,
•“View Full Profile”— links to a page with user’s profile information.
