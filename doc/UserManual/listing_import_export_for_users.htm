Listing Import and Export For Users

 Top  Previous  Next

The add-on "Listing Import and Export for Users" is not a part of iLister default package for distribution. It can be purchased separately for your iLister-based website for additional cost.

 

Listing Import and Export for users work very similar to Listing Import and Export for the administrator.

 

Who can import and export listings

User Groups have the option "Ability to Import and Export Listings". If the website administrator enables this option, all users of this User Group get the ability to import listings to their account at iLister-based website.

 

 

How to enable "Ability to Import and Export Listings"

It can be managed through Admin Panel: User Management > User Groups > $UserGroup > Edit > Ability to Import and Export Listings

 

How it works for users

Users can access the Listing Export and Import areas in the User Menu.

 

 

Import

•Import does not support the 'file' and 'video' field types.
•The listing fields which do not exist for a category will be ignored while importing data, the missing tree and list-type values will be left blank after the import. For example, the field "Price" does not exit for the category "For Rent", thus price values will be ignored when importing listing records in any case (either they are filled or not in the file). If you add an item with the "Navy" color, and this color does not exist on the website, the option "Color" will be left empty for this listing after the import.
•Column names in the importing file must be named exactly as the field IDs, except for fields of the 'Tree' type (StateProvince). For example, the 'Price' field has ID=Price, so use Price for the name of the corresponding column.
•The Tree type (as the 'StateProvince' field) should be defined as follows: in two columns, first is 'StateProvince[1]' - it is for the 1st level (State), second is 'StateProvince[2]' - it is for the 2nd level (Province).

You can see the examples of importing XLS and CSV files.

 

 

Export

The Listing Export is clear and intuitive. When specific category is selected, the export form starts to display all available fields for this category; otherwise, if no category is specified, only common fields are displayed.

Exported listing are collected to the archive export.tar.gz with the following structure:

•files
•pictures
•videos
•export_data.xls
