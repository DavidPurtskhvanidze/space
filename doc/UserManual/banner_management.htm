Banner Management Add-on

 Top  Previous  Next

The Banner Management add-on is not a part of iLister default package for distribution. It can be purchased separately for your iLister-based website for additional cost.

 

The Banner Management add-on (BMA) allows site administrator to easily add image and flash banners to WorksForWeb products-based website.

 

You can find the Banner Management section in Admin panel -> Content Management -> Banners.

 

How to Start

First you need to add a new Banner Group following the link "Add a New Banner Group" on the Banner Groups interface.

 

 

Banner Groups

You can define the following properties for a group:

•Group name — represents the ID of the banner group which will be later used to display banners
•Description
•Type – there are 3 types for the group: Flash, Static image, HTML/Javascript.
•Maximum size (Width and Height) of a banner in the group. The system ignores banner’s size if the "Width" and "Height" values are empty.

 

In case a group has at least one banner, you cannot delete the group. The editing is also limited; you can change the name of a group and its description only.

 

After a group is added, you can edit or add its banners through the "Edit & View" link.

 

 

 

Image Banners

You need to indicate the following settings in order to add an image banner:

•Caption
•Number of exposure - the number of time the banner is shown. When the number is reached, the banner is deactivated. There is no exposure limit if the field left empty.
•Lifetime (in days) - a number of days for which the banner is active. After this number of days the banner will be deactivated automatically. There is no lifetime limit if the field left empty.

In case both parameters (Number of exposure and Lifetime) are filled in, the banner will be deactivated when one of the parameters reached its limits.

•Link URL – the URL where a banner is linked to. Administrator can leave the field empty and in this case the banner will not be clickable and thus will not have any statistics.
•Delay time (in secs) – the time the banner is displayed in seconds.
•Image file – you can use the gif, png, jpg/jpeg file formats for the file upload. After the upload all files are converted to the png format.

Administrator can change the order of image banners arrangement by clicking the arrows "up" or "down" and edit any banner’s settings, and also expire or activate banners manually.

 

When banner is expired, administrator can activate it again and the BMA will ask to set new values for the "Number of exposure" and "Lifetime" parameters.

 

When view/editing image banner, site administrator can see additional information about the banner:

•Current file - displays current image file.
•Status - displays banner status: active or expired.
•Current number of exposure - displays current number of exposure.
•Current lifetime - displays current number of lifetime in days, hours and minutes.

 

 

Flash Banners

There is no "Delay time (in secs)" property for such banners, hits and clicks are not countered. Instead of the "Delay time (in secs)" property there is the "Number of exposure" parameter. Flash banners can have only one banner.

 

You need to specify the following properties in order to add a Flash banner (in a flash-type banner group):

•Caption
•Number of exposure - the number of time the banner is shown. When the number is reached, the banner is deactivated. There is no exposure limit if the field left empty. Flash banners rotate when a page is reloaded (refreshed).
•Lifetime (in days) - a number of days for which the banner is active. After this number of days the banner will be deactivated automatically. There is no lifetime limit if the field left empty.

In case both parameters (Number of exposure and Lifetime) are filled in, the banner will be deactivated when one of the parameters reached its limits.

•Flash file – you can use swf file format.

 

 

When view/editing flash banner, the administrator can see more information about the banner, the same as for image banner.

 

 

HTML/Javascript Banners

You need to indicate the following settings in order to add an image banner:

•Caption
•Number of exposure - the number of time the banner is shown. When the number is reached, the banner is deactivated. There is no exposure limit if the field left empty.
•Lifetime (in days) - a number of days for which the banner is active. After this number of days the banner will be deactivated automatically. There is no lifetime limit if the field left empty.

In case both parameters (Number of exposure and Lifetime) are filled in, the banner will be deactivated when one of the parameters reached its limits.

•Link URL – the URL where the whole text banner is linked to. All links located within the text banner will be deactivated if this parameter has a value. Administrator can leave the field empty and in this case all links inside will be active and clickable.
•Delay time (in secs) – the time the banner is displayed in seconds.
•Source – you can see the WYSIWYG editor built in. The administrator can use text, HTML or Javascript for this type of banners. Thus you can add a Google AdSense code and use it at the desired pages. To add a Javascript, please switch the editor from WYSIWYG to the Source mode.

 

Administrator can change the order of image banners arrangement by clicking the arrows "up" or "down" and edit any banner’s settings, and also expire or activate banners manually.

 

When banner is expired, administrator can activate it again and the BMA will ask to set new values for the "Number of exposure" and "Lifetime" parameters.

Statistics

Statistics is available for image banners only. Administrator can view statistics for each image banner following the link "Statistics". It can be found on the Image Banner Group page opposite each banner and at the "View/Edit" banner page.

 

Clicking the "Statistics" link the administrator will see the pop-up window with the following information:

•Banner report type - can be selected daily, weekly, monthly or yearly.
•Total clicks - the whole number of clicks on the banner
•Number of IPs - number of IPs (unique visitors who viewed the banner)
•The table with data:
1.Clicks - number of clicks on a banner from one IP
2.Visitors IP - the IP of a visitor
3.Date period - date of the click

 

 

 

How to display banners on the site

First of all, the administrator should find out which template displays the place on the site where he or she wanted to add banners. It can be easily done using the functionality of template identification.

 

Administrator can insert the code which displays banners to related template as explained below:

•Administrator should open the related template page in Admin Panel, and on the right side of the page the administrator will see the drop down lists of Modules and Functions.
•He or she needs to choose the "banners" module and the "show_banner_group" function underneath. Then administrator must specify the "group_name" parameter and click the "Insert" link above the "Module" dropdown list.
•Afterwards administrator will see a code which displays banners; it will look like the following:

{module name="banners" function="show_banner_group" group_name="top"}

•Administrator can cut the code and insert it where he or she wishes the banners to display, and use <center>, <align="right"> and other HTML tags to display banner group correctly.
