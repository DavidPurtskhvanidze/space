The functionality of iLister is organized in modules and functions. Each piece of functionality is represented as a function of a particular module.

Some modules are included by default to the initial iLister package, others can be installed separately and can be purchased for additional cost.

 

What Are Modules And Functions

Module is a set of data structures, routines, database tables, php scripts, php classes, templates, and Admin Panel interfaces that aggregates some of the iLister functionality and makes this functionality available as its function.

 

Module function is a piece of module's functionality that generates content on the website. Functions may have parameters that adjust the function behaviour.

For example the search_results function of the classifieds module generates the part of the page with the search results, the show_static_content function of the static_content module displays the static text block identified by the pageid parameter.

 

Available iLister modules are listed in Admin Panel -> System Configuration -> Module Manager, physically they are located at the folder /modules.

 

 

There are you can see 5 tabs with modules:

•Available Modules — contains the modules representing certain features for the website. You can manage these modules and perform all possible actions like enable modules, disable, upgrade or install.
•Sample Data Modules — are the modules which keep data for the initial iLister-based website configuration like the set of categories, listing fields, user groups, membership plan configuration etc. You cannot disable these modules.
•System Modules — are the modules absolutely necessary for the minimal website operation. You cannot disable these modules or manage them some other way from the Admin Panel.
•Module Upgrades — this section lists the modules available for the upgrade. You can see the module version installed at your website and a new version proposed for the upgrade.
•Addon Modules — these modules are not included into the default iLister package and can be installed separately for additional cost, if it is specified.

 

Developers can create their own modules for iLister product. In order to work with iLister, these modules must not require WorksForWeb license which is absolutely necessary for iLister operation. Thus such modules should be named with the prefix:

•free_
•custom_

How To Install A Module

Every time you add a new module (free or paid) to your iLister package you will need a new license, which you can get from the Tech Support or Sales Dept. by making a request.

In order to install a module you need to make the following steps:

1.get new licenses for your iLister-based website and upload them to the server;
2.put an archive with a module files to the folder /modules;
3.extract the archive;
4.set the 777 permissions for:
•/modules/new-module-name/apps/AdminPanel/files
•/modules/new-module-name/apps/AdminPanel/templates
•/modules/new-module-name/apps/FrontEnd/files
•/modules/new-module-name/apps/FrontEnd/templates
•/modules/new-module-name/apps/MobileFrontEnd/files
•/modules/new-module-name/apps/MobileFrontEnd/templates
5.clear the /cache/core folder;
6.go to Admin Panel -> System Configuration -> Module Manager;
7.install a new module by clicking the link "Install" opposite the new module name.

 

How To Upgrade A Module

Upgrading a module means that you get new/enhanced features provided by this module. Available module upgrades are listed on the tab "Module Upgrades".

The functionality related to this module and its child module will not work until the "Upgrade" action is performed.

 

Here is what you need to do:

1.put an archive with a module files to the folder /modules;
2.extract the archive, overwriting existing files;
3.set the 777 permissions for:
•/modules/new-module-name/apps/AdminPanel/files
•/modules/new-module-name/apps/AdminPanel/templates
•/modules/new-module-name/apps/FrontEnd/files
•/modules/new-module-name/apps/FrontEnd/templates
•/modules/new-module-name/apps/MobileFrontEnd/files
•/modules/new-module-name/apps/MobileFrontEnd/templates
4.clear the /cache/core folder;
5.go to Admin Panel -> System Configuration -> Module Manager;
6.install an upgraded module by clicking the link "Upgrade" opposite the module name.

 

Disabling A Module

You can easily hide unwanted features from your website by disabling the corresponding module. However, some manual work could be required. For example, when you disabled the module "Publications", you need to manually remove the corresponding publications section from the left menu.

Nevertheless, the overwhelming majority of modules do not require any additional manual adjustments for disabling.

How Modules And Function Are Used In iLister

There are two ways a module functions can be used in iLister:

1.As a main content generator function in page settings
2.As a generator of the page's part

 

The Main Content Generator

On the page editing interface the website's administrator is able to specify any function as a main content generator of the page.  In this case the output of the function will be placed into the mainContent div placeholder of the page's template.  In other words the content generated by the specified function will be "wrapped" by the page templates (which usually adds a header, footer and a menu to the page). Please also refer to the Page Building section.

 

However, if function specified in the page settings has the "plain output" flag set, then the function's content is not wrapped with a page template, but is sent to the user's browser as is. This flag is used for functions delivering binary content such as images or export archives.

 

Page Part Generator

To use module function for generating a part of the page website's administrator can put the following code into the page's template:

{module name="modulename" function="functionname" parameter1="value1" parameter2="value2" ... parameterN="valueN"}

where modulename, functionname, and parameterN="valueN" are replaced with the real module name, function name and parameter corresponding names and values. When page is generated by iLister this code will be replaced with the output of the function specified.

 

For example, the code {module name="static_content" function="show_static_content" pageid="about"} will be replaced with the actual static content block with id "about".

 

Using QUERY_STRING as Parameter for Function Call

If a {module name="...} call in template contains a parameter named "QUERY_STRING" then iLister interprets value of this parameter as a set of parameters encoded as query string. iLister extends parameters encoded in QUERY_STRING and passes them as if they were specified separately.

 

For example, the following call

{module name="classifieds" function="search_results" QUERY_STRING="action=search&default_sorting_field=activation_date"}

 

is equivalent to

{module name="classifieds" function="search_results" action="search" default_sorting_field="activation_date"}.

 

However, using QUERY_STRING parameter allows to specify array-like structures in parameters, for example:

{module name="classifieds" function="search_results" QUERY_STRING="action=search&default_sorting_field=activation_date&color[in][]=red&color[in][]=blue"}

 

The later call displays search results with listings having the "color" field set either to "blue" or "red".
