payments - the page of user payment management

 

Parameters

•payments  -  payments ID
•action  -  the action which should be done with payments  
•restore -  is set when values of form fields need be restored
•sorting_field - the field of sorting
•sorting_order - the order of sorting

 

Templates

•payment_form.tpl - the form for payments filter

Local variables

None

 

•payments.tpl - the table with the list of payments and available actions

Local variables

o$sorting_field -  the ID of a field which will be used for sorting
o$sorting_order -  the order of sorting
o$found_payments_ids  -  the IDs of the user payments made
o$payment_id - the ID of a payment
o$status - the status of a payment
o$isCompleted - contains the data is a payment was completed
o$total_price  -  the total price of all payments in the table
