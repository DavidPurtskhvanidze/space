Terms, abbreviation and acronyms

 Top  Previous  Next

Visitor - anyone who can browse your site but who is not logged in.

User - a registered and logged in visitor of the site.

Owner - a user who owns a listing.

Administrator - a person who has access to an Admin Panel and can manage iLister site, its content and objects (listings and users).

Front-End - a part of iLister which is visible to visitors.

Admin Panel (Back-End) - a part of iLister which is available for administrator only.

User Group - a group of users who share a specified set of features and resources of the site.

Trusted User - users who can activate their listings without administrator's approval and listing moderation.

Membership Plan - a model contract for user subscription for the site services.

Listing Package - a model contract for a single listing.

Credit - an abstract unit to make internal payments on iLister-based website which has the Credit System module enabled.

Listing - an advertisement.

Comment (or Listing Comment) - a note with opinion or a question to a listing.

Category - a set of properties (fields, templates and subcategories) which are united by logical attribute.

Field Type - a way of how listing information and data of users profiles are entered, stored and displayed.

Theme - a set of files (templates, images, css files and a configuration file) which define how iLister-based website is looking.

Template - a file used to separate content from the look in web design, and for mass-production of site pages. It is a basic component of a web template system.

Module - a set of data structures, routines, database tables, php scripts, php classes, templates, and Admin Panel interfaces that aggregates some of the iLister's functionality and makes this functionality available as it's function

Function - a piece of module's functionality that generates content on the website.
