Listing Reactivation

 Top  Previous  Next

iLister provides functionality to reactivate user's listings.

 

How it works

Reactivation is attempted by the task_scheduler function of the miscellaneous module which is run via external scheduler (e.g. cron) for those listings which have an option Listing Reactivation bought.

 

 

Listing reactivation

Listing Reactivation purchased for a single active listing by the users who have a contract, i.e. whose subscription is not expired.

 

Website notifies users about forthcoming listing expiration in accordance to specified number of days and if users enabled "Notify On Listing Expiration Date Approaching" option  at the "My Notifications" page.

 

To buy reactivation, a user should go to My Listings -> Manage Listing Options -> Listing Reactivation and select a Listing Package for Listing Reactivation among all available packages. A user can buy only one reactivation option for a listing.

 

In case the listing does not fit the conditions of the chosen listing package for reactivation, the website will apply the new conditions valid at the moment of Listing Reactivation purchase. For example, if listing has 6 pictures, but the package allows only 5 images, the excess 6th image will be deleted.

 

The listing will not be extended if the listing is in PENDING or REJECTED moderation status.

 

Either way the listing owner is notified via email that contains complete report on which of the listings were extended and which were not. This notification is sent to the user by the task_scheduler when the "Notify me on listing expiration" setting is set to "On" at the "My Notifications" page.

 

Administrator will receive a similar report on listing expiration and auto-extension but containing information about all users. Whether this report is sent to the administrator is controlled by the "Notify on Listing Expiration" check box in the system settings.

 

Administrator can turn off the Listing Reactivation feature by disabling the "Listing Reactivation Option" module in the Admin Panel -> Module Manager.
