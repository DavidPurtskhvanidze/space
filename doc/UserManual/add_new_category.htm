Listing Categories

Categories have hierarchical structure. The administrator can add unlimited number of subcategories. All specific fields of a category are relevant to listings in all subcategories of this category. In other words, a subcategory inherits all fields of its parent category.

 

For example, if you create "Salary" field for "Jobs" category, this field will inherit by all subcategories of "Jobs" category.

 

Listing fields can be relevant to either a particular category or to all categories. The former fields are called category-specific fields, while the later are called common fields. All final categories (the categories that do not have children categories) list all fields which are available for its listings.

The last column has two-direction arrows, which allows to change the order of fields via drag-and-drop. The field order is used to display listing fields on certain pages via the foreach cycle like Search Form, Listing Details page, Listing Comparison page. You can change the field order and, for example, move the field "Price" to the top to display this prices on the top of the Comparison Table and other pages that use filed order.

 

In iLister categories have treelike structure. You can add a top-level category which will appear on the same level as "Want-Ads", or you can add sub-categories of the unlimited depth. In order to add a sub-category, you need to click the parent category ID and follow the link "Add a New Subcategory to This Level". Further instructions are listed below.

 

You can find categories in Admin Panel -> Listing Management -> Categories.

 

 

 

How to Add a New Category

Please click on the "Add a New Category" link to add a category. If you like, you can specify templates which will be used for a new category. Below you can see that iLister categories do not have any specific templates by default.

 

 

If you like to use specific templates for a new category you must create them. You can use similar existing templates as a basis for new template files.

 

As you can see from the code shown below, iLister by default uses the foreach cycle to display category and system fields:

{foreach from=$form_fields item=form_field}

 {if $form_field.id == 'Video'}

         {if $package.video_allowed}

         <tr>

                 <td>[[$form_field.caption]]</td>

                 <td>{if $form_field.is_required} <font color="red">*</font>{/if} </td>

                 <td>{input property=$form_field.id}</td>

         </tr>

         {/if}

...

 {/if}

{/foreach}

 

Thus all new fields which you created for a category, or all common fields, will be added to the input form automatically.

 

You can learn how to identify a template (or several templates) whish are used on a page in the Template Identification section.

 

Category Templates

Category templates are specific for different categories. As long as categories have different set of fields, you need to use different files to define which fields can be applied to a category.

 

Input Form Template is used on the "Add a Listing" page.

 

 

Search Form Template is used to display an advanced search form for the category.

 

 

Search Results Template is used to display the information on the search results page for each search result.

 

Template for View Listing is used to display listing details and other listing-related controls. This template usually includes other templates which are responsible for displaying different parts of the "Listing Details" page.
