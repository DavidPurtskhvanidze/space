Import Listings

 Top  Previous  Next

The format of the file to import listings can be CSV or XLS.

 

The administrator can create a file with data for new listings and import it to the database:

•Define column names in the file to be imported in accordance with field IDs, except for fields of the 'Tree' type (StateProvince).

For example, the 'Price' field has ID=Price, so use Price for the name of the corresponding column.

•The column for listing category IDs should be named 'category'.
•For the 'featured' system field, set 'featured' in a column name (note all characters are lower case).
•For the 'pictures' system field, set 'pictures' in a column name (note all characters are lower case). The 'pictures' column should contain either full URL of a picture with 'http://' (http://www.YourSite.com/files/pictures/picture.jpg) or relative URL from the temp/import/ directory of the script (pictures/picture.jpg). Picture URLs should be separated with ";" (semicolon). In the CSV file, all listed picture URLs must be surrounded with double quotes: "pictures/picture_1.jpg;pictures/picture_2.jpg" - please refer to a sample CSV below.
•The Tree type (as the 'StateProvince' field) should be defined as follows: in two columns, first is 'StateProvince[1]' - it is for the 1st level (State), second is 'StateProvince[2]' - it is for the 2nd level (Province).
•Import is not supported for the File and Video field types.
•You can assign new listing to a user: set 'username' (all characters must be in lower case) in a column. The user must be registered on your site.

In order to import listings please do the following:

1.go to Admin Panel -> Import Listings
2.complete the 'System Import Values' form with the appropriate data:
•Category — choose a necessary listing category from the drop down menu. Imported listings will belong to the chosen type. If you want to import listings of several categories, please ignore thus parameter — each listing record from the importing file will be added to the specified category of this record. In this case the listing fields which do not exist for some categories will be ignored while importing data. For example, the filed "Price" does not exit for the category "ForRent", thus price values will be ignored when importing listing records in any case (either they are filled or not).
•Listing Package — choose a necessary package from the drop down menu. Imported listings will belong to the chosen package. Along with the Listing Package, you can choose which free listing features from the selected Listing Package should be activated for imported ads.
 

 
If no free listing features are available within the Package, you will not be able to activate any during the import.
•Check to activate all imported listings — this flag means that the imported listings will be activated after importation;
•Set activation date — the date of activation (the current date by default).
3.complete the 'Data Import' parameters:
•File — indicate a file to import.  A file can have the following extensions:
ocsv (Comma-separated values),
oxls (MS Excel),
ozip (zip archive), containing an .csv or an .xls file in the root of the archive,
otar.gz (tar gzip archive), containing an .csv or an .xls file in the root of the archive.
•File Type — choose the type of file to import. The type can be either with .csv or .xls extension.
•Field Delimiter — for CSV-files choose a field delimiter here (Comma | Tabulator | Semicolon);
•Not found values in DB will be — ignored or added to DB (only for values of the List or Tree type).

When you import an archive file, pictures must be in the root of archive in the /pictures directory.

When you import a .csv or .xls file, pictures must be copied to the /temp/import/pictures directory in the root of your iLister script.

 

After filling in all fields, click the ‘Import’ button.
