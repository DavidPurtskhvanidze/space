Autocomplete

The autocomplete allows visitors, users and the administrator to quickly find and select from a pre-populated list of values as they type, immediately examining the proposed search results and modifying current request.

 

 

How It Works

The autocomplete works on different search forms; it is implemented for such a fields as keywords, title, username and email.

 

The below example demonstrates how autocomplete works on the Seller Search form.

 

By default, the autocomplete displays up to 10 records, matching search criteria. This number is specified in the templates:

•/modules/field_types/apps/FrontEnd/templates/search/string_with_autocomplete.tpl
•/modules/field_types/apps/AdminPanel/templates/search/string_with_autocomplete.tpl

 

However, the administrator can change this number for the exact search form. For example, you can open a listing search form and find the line with the autocomplete search enabled:

{search property=keywords template="string_with_autocomplete.tpl" parameters=['element_id_prefix'=>'advancedSearch','preselection_fields'=>['category_sid']]}

 

You can change the parameters and add there the code 'max_items'=>20, so it will look like:

{search property=keywords template="string_with_autocomplete.tpl" parameters=['max_items'=>20,'element_id_prefix'=>'advancedSearch','preselection_fields'=>['category_sid']]}

 

Thus the autocomplete on this search form will show up to 20 records instead of the default 10.

 

Other parameters can be found in string_with_autocomplete.tpl.

 

 

Autocomplete Setting

The website administrator can turn on/off the autocomplete for keyword search by checking the option "Enable autocomplete for keyword search" in the Admin Panel -> System Settings -> General.

 

 

 

 

 

Partial Match

The partial match allows visitors, users and the administrator to quickly find listings by typing part of word.

 

How It Works

The partial match works on different search forms.

 

In order to insert it you need to specify "string.like.tpl" template for desired field:

{search property=keywords template="string.like.tpl"}
