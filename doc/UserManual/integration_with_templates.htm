Integration with Templates

iLister has specific format for text elements to recognize them as phrases.

 

You can put a text which you want to translate in double square brackets [[ ]] and the text will be ready for translation.

 

You can also force your iLister-based website to use any domain to translate a phrase. In order to do so you need to specify domain as follows within a phrase:

[[Miscellaneous!PhraseID]]

 

Please refer to the example below:

If you want to add a keyword search to a Quick Search form, go to Admin Panel -> Site Layout -> Module Templates -> edit “Classifieds Engine” -> edit “quick_search.tpl”

1.Add the following lines to where you want to locate this field :

<tr><td align="right">[[FormFieldCaptions!Keywords]]

       </td>

       <td>{search property=keywords}

       </td>

</tr>

2."FormFieldCaptions" is the domain (i.e. group of phrases) where "Keywords" belongs to. Square brackets '[[' and ']]' are opening and closing tags for a phrase that will be translated.
