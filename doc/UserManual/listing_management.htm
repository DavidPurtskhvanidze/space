Listing Management

 Top  Previous  Next

Manage listing interface of the Admin Panel allows administrator search, select listings, and perform action with the selected listings. To open the Manage Listings interface click on the "Manage Listings" link in the "Listing Management" section of the Admin Panel's menu.

The Search/Filter Form

On the top of the page there is a search/filter form. When you click on the form caption, the form collapses.

Using this form administrator can search for listings matching the criteria. The following search/filter criteria are available on this form:

•Listing ID — accepts any natural number, will select a single listing with the corresponding id (number).
•Category — will fetch only listings of the specified category.
•Listing Package — allows to include listings having certain listing package. The listing packages are listed grouped by membership plan they belong to. Besides all available listing packages this select box contains additional item "Deleted Listing Package". When this item is chosen, search will include only those listings, which packages do no longer exist in the system. For example, if the listing was created with listing package "sample" and after the listing had been created by the user site administrator deleted the listing package "sample", this listing will appear in the search results on the Manage Listings Admin Panel's page if the Listing Package control is set to "Deleted Listing Package".
•Activation Date — allows to set a range of dates. Only listings which activation date is within the range specified are found. Dates are accepted in date format set for default language.
•Expiration Date — same as above, but criterion is applied to expiration date.
•Username — allows to find listings belonging to the user with the specified username.
•Search listings posted by administrator — allows to find listings which belong to the website administrator.
•Featured — if checked, will select only featured listings.
•Moderation Status — allows to find listings with the specified moderation status (Pending/Approved/Rejected).
•Active Status — allows to find listings with the specified active status (Active/Inactive).

After filter/search criteria are entered into the form, administrator needs to press a "Find" button to get a list of listing matching the given criteria.

The Search Results

The search results are presented below the search form, split by pages. Administrator can select the number of listings displayed per page.

 

Per-listing actions

The search results interface contains controls that perform different actions, that can be applied to the listings found. Each search result line has the following action controls section. This section is revealed when the mouse is on the area of the specified listing:

•Manage Comments — opens a Manage Comments page with all comments related to the listing.
•Activation Control — if the listing is not active it displays "Activate" link, clicking which make the listing active,  if the listing is active it displays "Deactivate" link, which deactivates listing (without any additional prompt).
•Moderation Controls — displays "Approve" link if the listing not yet approved and "Reject" link, which always visible (allows to reject listing even if it is approved and active)
•Edit — opens an edit listing page in Admin Panel.
•Delete — deletes the listing from database permanently. If Javascript is enabled in the browser, then clicking this button produces the confirmation prompt pop-up dialog.

 

 

Mass actions with listings

Besides per-listing actions Manage Listings interface allows to perform mass actions with the listings found. The controls for batch execution of actions against the selected listings are located on the top header of the search results table.

Selecting listings for batch processing: Before any of the batch action can be used the administrator has to select one or more of listings, which will be included into the batch processing. To select the listings for a batch action administrator must click the checkbox located in the rightmost column of the search results table. There are also "Check all" and "Uncheck all" links are available for convenience. Please mind, however, that check/uncheck all links will select only those listings, that are visible on the current search results page, but not listings on the other pages.

You can also "Hide pictures" from the search results records if this is more convenient for you.

Executing the batch actions: iLister allows the following batch actions over the selected listings:

•Activate — makes all selected listings active and moderation status to "APPROVED", regardless of their current state.
•Deactivate — makes all selected listings inactive, regardless of their current activation state.
•Edit Packages — will open the listing package edit interface, allowing to adjust listing package setting for the selected listings only.
•Delete — will delete selected listings from the database (completely and unrecoverably).
•Reject — will set moderation status to "REJECTED"
•Assign Package — will change the listing package to the one that is selected in the drop-down box to the left of the button. The listing package is applied to all selected listings, but the activation and expiration dates are left intact.

After the button is pressed and corresponding action is applied to the selected listings the Manage Listing page will reload. As the result of the action applied, some of the selected listings will no long match the filter criteria and thus will not be displayed again. For example, if administrator finds all "PENDING" listing and clicks "Activate" on them, the page would reload without activated listings displayed because these listings no longer match the Moderation Status = "PENDING" criterion.

 

Listing Moderation

iLister has a system of listing moderation. This system is greatly enhancing the ability for the administrator to counter and combat the following problems:

•Illegal content,
•Spam,
•Listing Scams,
•Abusive, hateful, obscene and otherwise unwanted posts,
•Competitor resource advertising etc.

This is especially helpful for those website owners who are setting up free to list websites, preventing illegal or unwanted content to be displayed on the website and causing possible legal action towards the website owners.

The listing moderation system contains three key components: listing pre-moderation and post moderation, and user status update (trusted users vs. non-trusted users).

The listing pre-moderation system works for free and paid listings. It allows the administrator to screen all listings created by users and approve them prior for listings to go live.

There's also a post-moderation system that gives the administrator the ability to moderate listings after they have been edited or updated by the owner.

Important: By default, all listings are moderated; all new users are not trusted. To "turn off" the moderation system, you will need to set all users as trusted as they sign up. If you have any suggestions on how to improve this system, please contact Sales or Support with your particular suggestions.

The status of listings posted by site administrator is "Approved".

 

How It Works

When the Moderation system is activated, a newly created listing is not automatically activated. Instead, it gets the "Pending" status. It becomes active only after the administrator's approval. The administrator may "Approve" listings to accept them for publishing, or "Reject" the ones that he thinks require an update or re-formulation, or feature undesired content. The approved listings become active, whereas the rejected ones remain inactive.

 

The Moderation system is further enhanced by the Trusted User module. It gives the administrator the ability to set user as "Trusted Users".

Listing Moderation

                 

The current listing status is displayed in the Status field. The system has the following listings statuses:

•NULL - newly created listing, waiting for the payment to be made by user.
•PENDING – listing requires administrator's moderation, new or updated listing.
•APPROVED – listing approved by administrator. If listing is active (Active: 1) - it will be published on the site.
•REJECTED – listing rejected by administrator for any reasons. The owner of a listing has to edit/update such listing. After the update, the listing will switch to the "Pending" status.

The administrator may set any status to any listing, except for "Pending".

 

Moderation Status Display in the Front End

                 

Listing owner can view the current status of the listing on the "My Listings" page. The status is displayed by the search_results.tpl template.

If a listing status is either "Rejected" or "Pending", the listing owner can only update or delete the listing.

If a listing is approved by the administrator, the listing owner is also allowed to activate/deactivate the listing. Free listings are activated immediately, their activation date is set to the current, and the expiration date is set with respect to it. Paid listings can be deactivated by listing owners, but to reactivate such listings, the owner will have to pay for them.

If a listing is approved by the administrator, the system sets the current date as its activation date and sets the expiration date as well. The rejected listings' expiration dates are not changed after the administrator's approval.
