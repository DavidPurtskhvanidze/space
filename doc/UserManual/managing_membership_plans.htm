Managing Membership Plans

 Top  Previous  Next

The system provides you with a variety of options that the administrator can use to set membership plans. Membership plans will not have an effect until the administrator have assigned them to a User Group(s). In this section you will get known how to manage membership plans.

 

The ‘Membership Plans’ page of ‘Membership Plan Management’ section displays all of the existing Membership Plans that you currently have in the system. On that page you can add a membership plan by clicking ‘Add a New Membership Plan’ link.

 

When an "Add a new membership plan" page opens, administrator will see the following membership plan settings:

•Name — name of a membership plan;
•Description — description of a membership plan;
•Price — the price for subscription to a membership plan;
•Expiration Period — the number of days when a membership plan will be available;
•Type — select type of the membership plan. Please refer to the memberships plan overview for information in types of membership plans.
•Number of Listings — limitation of listing number which user who has a membership plan can add to the system. Leave it empty for unlimited listings amount.

 

To submit a new membership plan, click on the ‘Save’ button. The new membership plan will appear in the membership plans list.

To edit a membership plan, click on the ‘Edit’ icon in the membership plans list. The administrator can change all parameters of a membership plan except 'type'. A type of a membership plan can't be changed.

If there are no users in the system subscribed to a membership plan, then this membership plan can be deleted by the administrator. To delete a membership plan, click the ‘Delete’ icon in the membership plans list on the membership plans page.

 

Packages

On the ‘Edit Membership Plan’ page an administrator can add a new package. A package in a membership plan is a set of listing features, options and limitations which will be applied to a particular listing. A thorough description of package can be found in the overview section of this manual.

 

 

By clicking ‘Add a New Package' link you will see a dialog box ‘Add a New Package’ with the following fields:

•Name — name of a new package.
•Description — description of a new package.
•Price — price of a new package. This price will be charged user for listing activation.
•Listing Lifetime (Days) — listing lifetime before expiration in days.
•Number of Pictures Allowed — number of pictures a user can add to his listing.
•Is Video Allowed — this flag means that a user can upload video for listing. Not the same as YouTube video. This video is stored on the  iLister-driven website and is available under the "video" field type.
•Price for Upgrade to Featured – the price required for upgrading listing to a featured ad. If the price equals zero (0), the listings of this package can be made Featured for free.
•Price for Highlighted Feature — amount of money to charge users to make listing highlighted. If the price equals zero (0), all listings created/activated under the current Listing Package can become highlighted for free.
•Price for Slide Show Feature — amount of money to charge users to enable Slideshow options for their listings. If the price equals zero (0), all listings created/activated under the current Listing Package can have the image slideshow enabled for free.
•Number of Days for Sponsored Listing — the lifetime of the listing sponsored status (in days). The sponsored status does not expire in case it was activated by the website administrator. Empty value equals to 0 (zero).
•Price for Sponsored Listing — the price for sponsored feature. If the price equals zero (0), all listings created/activated under the current Listing Package can have the "Sponsored" status for free.
•Price for Video from YouTube Feature — amount of money to charge users to enable YouTube video options for their listings. If the price equals zero (0), all listings created/activated under the current Listing Package are allowed to have a YouTube video ID in a listing for free. Please see the YouTube feature description for more details.

 

Free listing features are available on the "Add a Listing" page and can be enabled there along with the creation of a listing.

In addition, users can turn these features on for existing listings on the "My Listings" page.

If no free listing features are available within the Package, users will not be able to activate any listing features for free while adding a listing.

 

To submit your package, click the ‘Save’ button.

The new package will appear in the ‘Packages Included’ table. By clicking the ‘Edit’ icon next to a correspond package you can change all the above-listed parameters of a package. By clicking the ‘Delete’ icon next to a correspond package you can delete a package.

 

Editing the package

When administrator click on the "Edit" button in the package list, a Edit Package opens with form pre-filled with current values of package's settings. Administrator can edit and save new values. However, the new values that administrator saved are not immediately applied to the all listings because listings retain their package settings until expiration and subscription type-contract retain all packages that were available at subscription. This is by design. The next time the package is used to create and activate a listing new values will take effect.

 

Synchronizing listing packages with membership plan packages

If for some reason administrator would like to apply new values in package settings to all listings immediately (although it may be unfair in some cases) there is an "Apply to the listings" button on the "Edit Package" page.

This link is available only if the package fields were not changed. If any of the package field were changed on the form, the "Apply to the listings" link becomes disabled. Administrator has to save the new values before applying them to the listings.

 

When you clicked this link, iLister tries to find all listings that were created with the current package and adjust the listings packages' settings according with new values of the membership plan package. This will not affect listings that use other packages. 

 

However, even in this case not all of the settings are applicable immediately. The reason is that the listing settings are checked only at certain events. For example the number of pictures allowed is checked only when user manages pictures for the listing. So, if there were 10 pictures allowed in the package when user created and activated the listings (and user has actually uploaded 10 pictures) and then administrator reduces this setting to only 6 pictures allowed and presses "Apply to all listings" then the exceeding 4 pictures will not be removed from the listing. However, if the user will try to deactivate and then activate the listing again, iLister will not allow to activate this listing again unless user either chooses another packages or removes the excessive pictures. Similarly, if user has paid $10 to enable slideshow, but then administrator set the price for slideshow to 7 and pressed "Apply to the listings" then this won't give the user $3 back. However, those users that have not yet enabled slideshow, will have to pay only $7.

 

The "Apply to the listings" button as well appears in the table of packages included in the membership plan edit page. It has the same effect as the button on the edit package page.

If the membership plan is of "Subscription" type, it's packages setting are copied into the users' contracts (see more on this in overview). Thus, when package settings are changed by the administrator, the new settings are applicable only for new subscriptions (after user registration or after previous subscription expires). To force the propagation of the changes to the existing user contracts, administrator can use the "Apply to user contracts" button.  After pressing this button all the current settings of the membership plan's package will be applied to all user contracts that contain the corresponding package.
