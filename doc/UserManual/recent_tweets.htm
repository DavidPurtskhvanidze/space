Description

The Recent Tweets module allows to display latest tweets from the Twitter on the iLister-based website.

 

Recent Tweets Settings

You tweet source is configurable in Admin Panel -> System Configuration -> System Settings -> Social Networks -> Recent Tweets Configuration:

•Screen Name — is the name of the tweet source. By default it contains the WorksForWeb Twitter name.
•Cache Lifetime (in minutes) — the lifetime for Twitter cache. It is necessary to use the cache in order to minimize the possibility of exceeding the Twitter Rate Limit of 150 units. When the Rate Limit is exceeded, the Twitter does not allow to display tweets. In this case iLister displays outdated tweets from the cache if  there are any; or, in case there are no tweets in the cache, the website shows the related notice: "Rate limit exceeded. Clients may not make more than 150 requests per hour."

 

You also need to specify the twitter Consumer Key, Consumer Secret and Access Tokens in Admin Panel -> System Configuration -> System Settings

-> OAuth & OpenID. You can read more about it in Facebook, Twitter and OpenID Login section.

 

 

The Recent Tweets Module

The look of the recent tweets section can be easily changed in the design.css file at /application/modules/recent_tweets/apps/FrontEnd/templates/_files/design.css

The birds can be modified in /application/modules/recent_tweets/apps/FrontEnd/templates/_files/tweet.png

 

You can find the module at Admin Panel -> System Configuration -> Module Manager among other modules. There you can disable the Recent Tweets module if you do not want to use it.

 

In order to insert the recent tweets section to a page (template), please use the following code:

{module name="recent_tweets" function="display_recent_tweets" count="3"}

 

The "count" parameter specifies the number of tweet records displayed.
