iLister uses Smarty v.3 as a template engine to display HTML content. Smarty facilitates a manageable way to separate application logic and content from its presentation. You can learn more about Smarty on its official website.

 

Smarty Functions

iLister uses the following Smarty functions:

•url - specifies path to files. iLister uses it to connect to CSS files and display images. If the file does not exist in the current theme, the system takes it in a parent theme.
oParameters:

file - the name of the file and the path from the root of a theme to the file

oExamples of use:

<img src="{url file='main^logo.png'}" alt="Your Logo Here" />

 

•require - connects external component in templates.
oParameters:

component - the id of external component (the name of the folder in the directory of external components; by default it is system/ext - you can change it in default.ini EXT_LIBRARY_DIR)

file - the file for connection (a relative path from the root of external component to the file)

oExamples of use:

{require component='jquery' file='jquery.js'}

{require component='jquery' file='css/basic.css'} 

When you connect a component which depends on another component you first need to connect a "parent" component. This is also fair for jquery plugins: you first need to connect jquery.js library before connecting a plugin.

 

•module - launches module function in templates.
oParameters:

name - module name

function - function name

QUERY_STRING - parameters in QUERY_STRING

additional parameters processed by function (e.g. form_template parameter for classifieds:search_form function)

oExamples of use:

{module name="classifieds" function="search_form" form_template="quick_search.tpl" category_id="HomesForSale"}

{module name="classifieds" function="browse" level1Field='State' browse_template='browse_homes_by_state.tpl' category_id='HomesForSale'}

{module name="classifieds" function="search_results" action="search" QUERY_STRING="username[equal]="|cat:$user.username} 

 

 

Smarty Blocks

You can see the following Smarty blocks in iLister:

•title
•keywords
•description

 

They corresponds with HTML tags <title> and <meta>'s elements "description" and "keywords".

 

They are specified in the settings of Site Pages in Title, Description and Keywords parameters.

 

You can use them as follows in templates:

<meta name="keywords" content="{$KEYWORDS}" />

<meta name="description" content="{$DESCRIPTION}" />

<title>My iLister site{if $TITLE ne ""}:&nbsp;&nbsp;{$TITLE} {/if}</title>

 

 

Smarty Global Variables

Global variables are available for any page and can be used in any template.

•$GLOBALS - associative massive containing:
osite_url - the URL of site specified in the constant "SITE_URL" in apps\FrontEnd\config\local.ini and apps\AdminPanel\config\local.ini
ouser_page_uri - uri of the current page
ocurrent_user - current user information
▪logged_in keys defines is a user is authorized and in this case you can also use the data like username, email etc.
othemeInheritanceBranch - hierarchy of current theme inheritance. It is used to connect CSS files of the whole inheritance branch.
oradius_search_unit
ogoogle_maps_API_key
onotification_email
oenable_wysiwyg_editor
ocustom_settings - array containing all settings from Admin Panel -> Custom Settings.
•$TITLE
•$DESCRIPTION
•$KEYWORDS

 

You can use global variables as follows in templates:

{if $GLOBALS.current_user.logged_in}

<div class="LoginBlock">

<b>[[Welcome]], {$GLOBALS.current_user.user_name}</b>

</div>

{else}

<div class="LoginBlock">

 <span>

         <a href="{$GLOBALS.site_url}/login/">[[Sign in]]</a>

 </span>

 <span>|</span>

 <span>

         <a href="{$GLOBALS.site_url}/registration/">[[Register]]</a>

 </span>

</div>

{/if}

 

 

Other Smarty Variables

Each page has more variables available other than global variables. You can easily learn them using Smarty {debug} tag: insert it to a desired template in the very beginning and save it, and then open the page which uses this template in a browser.

 

Page Building

Using Site Page configuration, iLister:

1.launches Site Page function and passes parameters to it (e.g. title, template, module, function and its parameters, keywords, description).
2.finds a template for this Site Page and inserts to the mainContent div the data from module, function and its parameters, which specified is the settings of the Site Page;
3.starts the calls of Smarty variables, blocks, modules and functions.

Themes

A theme defines the look of iLister-based website. Theme is a directory containing a set of files - templates, images, css files and configuration file.

In fact, themes are modules.

The version of a custom theme module equals to the current product version plus module version, e.g. "6.0.0-1".

You can find theme module folders at /module/theme_$themeName.

These theme subfolders contain only those modules, which templates are redefined by the current theme.

 

iLister system uses the name of theme's directory as theme's name which is displayed in Admin Panel -> Themes.

 

 

Theme Inheritance

In order to get rid of file duplication iLister uses theme inheritance system; thus, you can create a theme based on one of existing themes.

 

The inherited theme called "parent", which also can be inherited from a third theme (hierarchic inheritance).

The inheriting theme called "child".

 

In order to change a part of a child theme you need to make a copy of corresponding template from the parent theme and modify it. The same works with images: if you want to use another image (logo, icons etc.) in a child theme you need to place this image file to the child theme directory. A relative path from the root of child theme to the template or image must be the same as used in the parent theme.

 

In order to manually change theme inheritance you need to specify its name in the "parent" field in the database table core_themes for a child theme record and also in the file /modules/$childTheme/config.ini by changing the parameter "inherits".

 

Base Templates

The base templates used for inheritance by all themes are located in the /templates folder of each module, e.g.

/modules/listing_comments/apps/FrontEnd/templates.

 

This folder has all templates and images necessary for iLister operating. Also the iLister product has its own product-specific base theme called "iLister_base" which can override base templates of a module, if necessary.

 

 

Theme Management

You can create themes based on existing "base" themes.

 

You need to follow the below steps in order to create a new theme:

1.go to Admin Panel -> Site Layout -> Themes
2.define which of the existing themes will be used as a parent theme;
3.create a new theme using the provided form.

 

Afterwards you can use child theme and it will repeat the look of parent theme.

 

In order to change a part of the content (text or image) you need to define which template provides it. Then copy this template or image from parent theme to child theme leaving a relative path intact and afterwards modify template or image as desired.

 

 

 

 

 

Template Identification

 

In order to identify the template which provides a content for a page:

1.open the source code of the page;
2.find there the content which you want to change;
3.find the comments that specify the beginning and the end of the template which provides this content:

<!-- START $moduleName:$themeName/$templateName -->

... 

<!-- END $moduleName:$themeName/$templateName -->

 

where:

•$moduleName - directory name in the theme
•$themeName - theme name
•$templateName - template file name

 

 

Images and Style Sheets

Images and style sheet files are stored at the module-related folder like /module/$ModuleName/apps/FrontEnd/files/$ThemeName

For example, /modules/main/apps/FrontEnd/files/iLister_base

 

Some modules do not have the folder /file, and you can create it yourself if you need to add new custom image or style sheet there.

 

You can use the following construction in template files to display an image:

<img src="{url file='main^logo.png'}" alt="Your Logo Here" />

Please note how the path to an image is specified for the <img> tag.

 

You can connect a CSS file to a template as follows:

1.use the @import rule which allows you to include another style sheets to your default CSS file (design.css):

@import url("new-css-file.css");

2.connect a CSS file with HTML's "link" element:

<link rel=StyleSheet href="style.css" type="text/css" media="screen, print">
