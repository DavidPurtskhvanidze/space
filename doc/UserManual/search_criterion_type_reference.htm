Search Criterion Types

 Top  Previous 

Below is the list of criterion types currently supported by iLister and brief information on usage of each criterion type. The description of how criteria are used in iLister can be found on the dedicated page in the Basic Concepts.

 

equal

The equal criterion type is a single-value criterion.

 

Example:

 State[equal]=Oregon

 

This criterion demands that the value in the State field has the value of Oregon.

 

geo

The geo criterion type is a multi-value criterion. It has two keys:

1.location — a name of the known geographic location (managed on the "Geographic Location" page of the Admin Panel);
2.radius — maximum distance from the specified location to search within. Units for the distance are set in System Settings (either miles or kilometers)

 

Example:

 Zip[geo][radius]=100&Zip[geo][location]=90001

 

This criterion demands that the search results may have the values in the Zip field which are not farther than 100 distance units (miles or kilometers) from the location of 90001.

 

 

in

This is a multi-value criteria, however it does not have any keys. One can specify multiple values for this criteria by adding an empty square brackets, i.e. field[in][]=1&field[in][]=2.

 

Example:

 Color[in][]=Green&Color[in][]=Blue

 

Criterion evaluates to true if value of the field matches any of the values specified in the criteria. (the color is either "Green" or "Blue" in the example above).

 

 

includes_all

The includes_all criterion type is a single-value criterion.

 

Example:

 keywords[includes_all]=park+lake+playground

 

This criterion demands that the search results must contain all specified keywords, i.e. park and lake and playground.

 

 

includes_any

The includes_any criterion type is a single-value criterion.

 

Example:

 keywords[includes_any]=park+lake+playground

 

This criterion demands that the search results may have any of the specified keywords, i.e. either park, or lake, or playground, or all of them.

 

less

This is a single-value criterion.

 

Example:

 Baths[less]=3

 

This criterion demands that the value in the Baths field has the value less than 3, i.e. 2, 1 or 0.

 

 

like

This is a single-value criterion.

 

Example:

 Description[like]=new+sport+car

 

This criterion demands that the value in the Description field contains the string "new sport car".

 

 

more

This is a single-value criterion.

 

Example:

 Doors[more]=2

 

This criterion demands that the value in the Doors field has the value more than 2, i.e. 3, 4 or higher.

 

 

not_earlier

The criterion type not_earlier is a single-value criterion and does not expect any keys. It is used for the fields of the "date" type only; iLister has only two system fields of the type "date": activation_date and expiration_date.

 

Example:

 activation_date[not_earlier]=05-21-2010

 

This criterion demands that the search results may have the values in the activation_date field of 05-21-2010 and later.

 

 

not_empty

The not_empty criterion type is a single-value criterion and does not expect any keys or value. When this criterion type is present in a query string, it tells the site to search for any value for the specified field.

 

Example:

 Rent[not_empty]=

 

This criterion demands that the value in the Rent field has any value: 0, 1, 87000, 0.99 or other.

 

 

not_equal

The not_equal criterion type is a single-value criterion.

 

Example:

 State[equal]=Utah

 

This criterion demands that the value in the State field can have any value excepting "Utah".

 

 

not_later

The criterion type not_later is a single-value criterion and does not expect any keys. It is used for the fields of the "date" type only; iLister has only two system fields of the type "date": activation_date and expiration_date.

 

Example:

 expiration_date[not_later]=12-31-2009

 

This criterion demands that the search results may have the values in the expiration_date field of 12-31-2009 and earlier.

 

 

not_less

This is a single-value criterion.

 

Example:

 Beds[not_less]=2

 

This criterion demands that the value in the Beds field has the value not less than 2, i.e. 2, 3 or more.

 

 

not_more

The not_more criterion type is a single-value criterion and does not expect any keys.

 

Example:

 Engine[not_more]=5

 

This criterion demands that the value in the Engine field has the value of 5 and less, i.e. 5, 4, 3 and less.

 

 

tree

 

The tree criterion type is a multi-value criterion. It expects numbers of level as keys and ids of tree nodes and values.

 

Example:

 MakeModel[tree][0]=291&MakeModel[tree][1]=294

 

This criterion demands that the value in the MakeModel field has the value of 294, while the value's parent should be 291. It looks pretty lame for a human, but it allows for a quicker search restore. Since most of times these criteria are generated by a {search ...} tag in the templates it does not make any additional complexity for the website administrator. Nevertheless, we are working on brining more human-readable criteria for the tree-typed fields which should facilitate usage of tree-based criteria in links and external forms.
