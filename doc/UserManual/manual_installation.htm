Manual Installation

 Top  Previous  Next

Below are instructions on installing the iLister  manually. Use these instructions if your hosting environment is not typical (i.e. self-hosting) and in case when installation script stalls or cannot proceed due to incompatibility with hosting environment.

 

In order to complete the manual installation you need a FTP of shell (via SSH,) access to the server. Cpanel or Plesk can be used as well. The server should have unzip a software installed (any tool that is capable or unpacking a ZIP archive file).

 

Here is what you need to do:

1.upload the archive with iLister to your server with the help of FTP client or SSH client (i.e. scp tool); If you do not have ability to run an archive on you hosting server, then you will have to unpack the archive and upload all the files to the installation location preserving directory structure. (Uploading of 1000+ files can take a substantial amount of time, even on a fast internet connection).
2.Open a shell and change directory to the installation location (where the uploaded archive is). You can use PuTTY if you connect from a Windows computer to a UNIX-like computer. You will need the username and password assigned to you by your hosting provider.
3.Extract the archive (skip this step if you have uploaded an unpacked distribution): On most UNIX-like operating system the following comand will unpack the archive to the current directory: unzip -aL iLister Archive.zip.
4.Create a database for iLister and a user that can connect and use this database.
5.If you have a Cpanel or other tools to do that you can use them. Please refer to the user manual of your tool of choice on doing this. Instruction below will allow you to create a database using a mysql command line tool that is included into MySQL server distribution package. First you need to login into MySQL server using your own username and password instead of the words in capitals:

mysql -u USERNAME -pPASSWORD

Please note that there is a space between -u and an username but there is no space between -p and password.

Then execute the following command. Please make sure each command is terminated by a semicolon. Type the command and press ENTER after each command.

create database DBNAME;

grant all on DBNAME.* to USER identified by "USERPASSWORD";

The later command will create user (if it was not existing before) and grant all possible privileges to all tables in the database to the specified user. This will as well set user's password to USERPASSWORD. If you adding privileges to an existing user and would like to avoid user's password being reset then omit the "identified by ..." clause and terminate the command (with a semicolon) right after the username.

Then type "exit" to exit the command line client.

6.Set permissions to allow apache server to be able to write to the following directories (relative to the root of software installation):
•/cache
•/modules
•/languages
•languages/*
•/files
•/files/*
•files/*/*

This step is usually not necessary if you are installing iLister on MS Windows. If you have a shell access to your UNIX/Linux hosting server, then you can use the following command to set the permissions:

chmod a+w cache/ modules/ languages/ files/ files/* files/*/* languages/*

Run this command from in the root of the iLister installation directory.

If you do not have a shell access you can use FTP client to change the permissions. To do so use your favorite FTP client to login to your server and change permissions to 777 to the folders and subfolders and 666 to all their files in the folder listed above.

7.Open the file apps/FrontEnd/config/local.ini for editing and paste the following into it:

DBHost = localhost

DBName = DBNAME

DBUser = USER

DBPassword = USERPASSWORD

MySQLCharset = utf8

SiteUrl = http://yoursite.com

Replace the DBNAME, USER, USERPASSWORD with the real database name, username and password. Replace http://yoursite.com with the real URL of your website. Please make sure that there is no ending slash "/" in the URL. Save the file.

8. Edit the apps/AdminPanel/config/local.ini  in a similar manner.

DBHost = localhost

DBName = DBNAME

DBUser = USER

DBPassword = USERPASSWORD

MySQLCharset = utf8

SiteUrl = http://yoursite.com/admin

Please note that for the Admin Panel the SITE_URL ends with a "/admin"

9. Generate a trial 15-day license for your site or use a permanent license which you received from the Tech Support Dept. and upload it to your server at the root of iLister i.e. /license.
10.Afterwards, please delete the install.php file located on your server at the root of the installation.
11.Then open the Admin Panel and wait until it installs the modules which are required for the minimum website operation. This installation will be performed automatically.
12.When the installation is completed, please log in to the Admin Panel and install other modules which you plan to use at your iLister-based website at Admin Panel -> System Configuration -> Module Manager.
