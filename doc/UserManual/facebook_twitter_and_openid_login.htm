Overview

Visitors can log in to iLister-based website with their Facebook, Twitter, Google or OpenID account and get instant access to the features of the website.

 

What is OpenID

OpenID is an open standard that describes how users can be authenticated in a decentralized manner, avoiding the need for services to provide their own systems and allowing users to consolidate their digital identities. You can learn more about OpenID on the official website or Wikipedia page.

 

OpenID authentication is now used and provided by several large websites. Providers include AOL, BBC, Blogger, WordPress, IBM, MySpace, PayPal, VeriSign, LiveJournal, Yahoo! and others.

 

Thus a person with existing Yahoo! account can easily log in to any iLister-based website which has this feature.

 

Facebook, Twitter and Google

Facebook, Twitter and Google use OAuth (Open Authentication) — the service that is complementary to, but distinct from OpenID. You can learn more about OAuth on the official website or Wikipedia page.

 

OAuth — an open standard for authentication. It allows users to share their private resources (e.g. photos, videos, contact lists) stored on one site with another site without having to hand out their credentials, typically username and password.

 

How to Turn On Facebook Login

In order to enable Facebook login, site administrator should register his site (so called "Create App") on Facebook and get facebook application ID and application secret.

 

One can do this at https://developers.facebook.com/ following the steps specified there.

 

 

 

When the app is created and the site has been registered, you need to "Add Platform" in Settings:

 

 

Then you need to specify the URL of your website:

1.site URL is the full URL of your iLister-based website (e.g. http://www.my-site.com);
2.Mobile Site URL is the full Mobile URL of your iLister-based website (e.g. http://www.my-site.com/m);

 

 

 

How to Turn On Twitter Login

In order to enable Twitter login, site administrator should register his site ("Register an Application") on Twitter and get twitter consumer key and consumer secret.

 

You can register the website at https://apps.twitter.com/ following the steps specified there.

 

 

Please specify the the "Callback URL" parameter: your website URL + /user/openid_oauth_login/

e.g. http://www.my-site.com + /user/openid_oauth_login/ = http://www.my-site.com/user/openid_oauth_login/

 

When the application is created and the site has been registered, you are redirected to the twitter application page of your website where you can get twitter Consumer Key, Consumer Secret and Access Tokens.(API key = Consumer Key, API Secret =Consumer Secret)

 

 

Then you need to check if your type of access is "Read and Write".

 

 

How to Turn On Google Login

In order to enable Google login, site administrator should register his site ("Create Project") on Google and get Google Client ID and Google Client secret.

You can register the website at https://console.developers.google.com following the steps specified there.

 

 

In the left sidebar, select APIs & auth. A list of Google web services appears. In the sidebar, select Credentials.

In the OAuth section of the page, select Create New Client ID.

 

Please specify the the "Authorized Redirect URI" parameter: your website URL + /user/openid_oauth_login/

e.g. http://www.yourwebsite.com + /user/openid_oauth_login/ = http://www.yourwebsite.com/user/openid_oauth_login/

 

In the sidebar, select Consent screen.

 

In the resulting Credentials section, copy the Client ID and Client secret that your app will need to use to access the APIs.

 

 

OpenID and OAuth Settings in Admin Panel

Site administrator should complete the OpenID and OAuth settings in Admin Panel -> System Settings.

 

Default User Group — is the User Group for the users who are signed in with OpenID or OAuth providers. The User Group "Buyers" has no Membership Plan, and thus their users can search listings, setup notifications, add listing comments and cannot post ads. Later these users can change their User Group if they want to post listings.

 

If no User Group was selected for the third-party login, the website would not display the third-party login section in the Front-End (please refer to the screenshot below to see the section under the usual "Sign In" form).

 

 

How It Works for Users

Everywhere a visitor needs to become a user, i.e. log in to the iLister-based website to post a listing or add comments, a visitor prompted to login using third-party accounts of OpenID or OAuth providers.

 

When visitor allows your website to use the OpenID or OAuth account information, the website creates a real user in the User Group specified in the Default User Group parameter in Admin Panel -> System Settings -> OpenID & OAuth.

 

The username for third-party registration is configured with the prefix "ext_" and 8-symbol alpha-numeric part.

 

Users can change this auto-generated username and create a password on the My Profile page.

 

After the registration, users are able to log in using both: their OpenID / OAuth accounts and the internal website username (auto-generated or already changed username).
