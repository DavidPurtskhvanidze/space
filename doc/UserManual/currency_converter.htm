The Listing Currency Converter module is an add-on and is not a part of iLister default package for distribution. It can be installed separately for your iLister-based website. Please request the installation conditions from our Sales Dept.

 

Overview

Listing Currency Converter gives the ability website visitors to view listings with selected currency. The administrator can specify currencies which can be selected by visitors.

 

 

Currency Converter Settings

The conversion service is provided by https://openexchangerates.org

 

In order to use the Currency Converter module, the website administrator should create an account at https://openexchangerates.org/signup/free. This service allows to get a free account for all currencies available on the service with the limit of 1000 requests per month. Please check the actual service conditions at the official Open Exchange Rates website.

 

After you created an account and specified the website where you plan to use rate service, you'll be given the application ID.

 

You should insert it to the Admin Panel -> System Configuration -> System Settings -> Listing Currency Converter -> API Code

Also you need to specify:

•Listing Main Currency — the currency which is used by users when they create listings on the website.
•List Of Available Currencies — the list of currencies which are available for conversion and their rate to listing main currency. Please note that currency rates are reloaded once an hour. Administrator can see all currencies, but only active ones will be visible to the website visitors in the Front-End.
•Show Decimal Part For Converted Money-type Fields — when enabled, it shows decimal separator and decimal part for all prices in accordance to language settings.

 

If you plan to activate the currency which has a high rate to a main listing currency, you can face the situation when converted prices are too large and do not fit into default design of your website. Here is what you can do:

•disable the option "Show decimal part for converted money-type fields" — this will cut off the decimal separator and decimal part for all prices, and thus will save you some space.
•adjust the styles in related design.css files, related to the section which displays a price, or the style of a money-type filed itself. For example, you can edit the size of the font (font-size) and its weight (font-weight), and instead use another color (color) to highlight the price information among other fields.

 

How It Works For Visitors

Website visitors can choose current listing currency among active currencies specified by the administrator.

 

After it was saved, a new listing currency will be applied to every money-type listing field.

 

Search forms will display two price-related fields: the active one (Price EUR) will relate to selected currency, while inactive (Price) will automatically calculate price from visitor currency to main listing currency.

 

Search results will display price in selected currency, but the Listing Details page will show both prices: in main currency and the converted one.
