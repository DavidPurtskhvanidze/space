Internationalization and Languages

 Top  Previous  Next

You can translate both the front-end and the Admin Panel of a iLister-based website to any language with a built-in interface for internationalization.

Front-end and Admin Panel use the same language files to avoid duplication of similar phrases and their translations.

 

Each language has its own settings which are considered for the front-end and back-end display:

•Language ID — please use a two-letter language ID (en, fr, es). You can see a full list of IDs in ISO_639-1.html
•Language Caption — this caption will be seen in the "Select Language" drop-down on the front-end of your site.
•Active — you need to set a language as “active” if you want visitors to see your new language on the front-end. Otherwise it will be visible only in Admin Panel.
•Front-End Theme — you can specify one of the existing themes on your site for this exact language. For example, it is very convenient to use a Right-To-Left theme for Arabic language, and for other languages (as English, Spanish or Chinese) use another theme with Left-To-Right layout.
•Admin Theme — you can specify one of the existing Admin Panel themes on your site for this exact language. Admin Themes can be connected with languages like Front-End and Mobile themes.
•Mobile Theme — you can specify one of the existing mobile themes on your site for this language. When users switch language on the Mobile Front-End of your site, they will see the theme which is defined for this language.
•Date Format — languages can display dates in different formats.
•Decimal Separator — you can select decimal separator that is normal for this language.
•Thousands Separator — you can also set thousands separator standard for a language.
•Decimals — you can specify the number of decimals to display after decimal separator. Please note that this setting will overwrite the setting "Signs number after comma" in all float-type fields.

 

 

How to add a new language

 

1.Go to the Language Management -> Manage Languages section in your Admin Panel.
2.Click “Add a New Language”
3.Enter Language ID, Language caption and fill out other fields if needed

NOTE: If you want users to see your new language on the front-end, you’ll need to set it as “active”. Otherwise it will be visible only in Admin Panel.

4.Click “Add” button

 

The new language will be added to the system. Now you can translate you site to this new language.

 

CAUTION: To add a new language, use ONLY the method described above. No other ways, such as manually copying and renaming language files, should be used to guarantee proper system operation.

 

 

Translating your site to other languages

There are two ways to translate your site to a new language. Below you can find descriptions of both of them.

 

Translation via Admin Panel

1.Go to the Language Management -> Manage Phrases section in your Admin Panel.
2.Choose your language and domain(s) in the form.

NOTE: Phrase domain is a category of phrases, which determines the context of the phrase. You can choose the domain you need to filter phrases, or you can choose “Any” domain, in the case all phrases will be displayed, grouped by phrase domains.

3.Click “Show” button.
4.Click on the Phrase ID you are going to translate and enter the translation.
5.Click “Save”

 

 

Translation via Front-End or Back-End Highlight Mode

1.Go to the System Configuration -> System Settings section in your Admin Panel
2.Set the “Mark Phrases That Are Not Translated” setting to “highlight”
3.Go to the front-end (or stay in Admin Panel if you plan to translate its phrases) and choose your language in the “Select Language” select box.
4.All phrases that are not translated will be displayed with a red “(T)” symbol at the end. By clicking on the (T) sign, the translation window will be opened. You can enter the translation to the form and click Save.

 

 

NOTE: You’ll need to enter you Admin Panel access credentials when trying to translate anything for the first time.

 

5.Go to the System Configuration -> System Settings section in your Admin Panel
6.Set the “Mark Phrases That Are Not Translated” setting to “default”
7.Turn off the “Mark Phrases That Are Not Translated” option.

 

After translating all phrases, you can have your users choose appropriate language from the list of available languages.
