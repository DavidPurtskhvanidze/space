General Information

 

The browsing feature allows your visitors to browse listings within a particular listing category or listing field. Browsing offers the following functionality:

•Administrator can specify Browsing Settings for each category, including root category.
•Administrator can set up browsing pages, configure browsing levels: which category should be displayed and what listing fields are available for browsing.

Note: for browsing you can use the fields of string, list, integer, geo and tree types only.

You can use the fields of tree-type several times - in accordance to the number of levels of the specific tree-type field. When you specify it for the first time, iLister will browse by the 1st level of this field.

For example, you can specify: State, MakeModel, Condition, MakeModel. In fact it will browse and display by State, Make, Condition, Model.

•Administrator can display browsing section in templates using the following code construction:

{module name="classifieds" function="browse" category_id="root" fields="State" browse_template="browse_by_state.tpl"}

•Visitors are able to browse within specified browsing levels as set up by the administrator.
•Browsing results are a list or listings found with brief descriptions (additional details can be added by the administrator via templates)

 

Each category has Browsing Setting which can be specified by the administrator in Admin Panel -> Categories -> edit a category -> follow the link "Edit Category Settings".

 

 

 

When configuring Browse Settings for a category, please be aware of the following:

1.You can use category fields or the fields of parent category(categories) as well as sub-categories of the current category for setting browsing levels. The field for sub-categories is category_sid.
2.You can add levels to the "Browsing Settings" form by clicking the button "Add browsing level". All available fields for the current category will be listed in the form "Available browsing fields" along with category_sid field for sub-categories. Then you'll be able to change the order for browsing levels using navigation buttons Up ↑ and Down ↓, or you can delete excess levels.

 

 

Administrator can set up as many browsing pages as desired. In order to create a browsing page, create a new Site Page with a "browse" function of the "classifieds" module.

 

 

When setting up a browsing page, please pay attention to the following:

1.The browsing category is set up in the parameter "category_id". In order to browse within all top-level categories, you need to place "root" to the category_id parameter.
2.The levels for browsing are specified in the parameter "fields" and delimited by comma ",". You can use either listing fields available for this category or its parent categories, or a subcategory which defines as category_sid among fields for browsing. For example, you can put the following:
•category_id: For-Sale
•fields: State, category_sid, NewUsed

In the above example visitors will see:

1)browsing page with list of States. Clicking on a State, visitor will see...
2)browsing page with sub-categories of the "For Sale" category. When visitor clicks a link with sub-category (e.g. Books), he will see...
3)a page with browsing by "NewUsed" listing field. Following one of the links, a visitor can see either browsing results (listings) or next browsing level based on the Browsing Settings of the selected category (i.e. Books from the above example).
3.You can also specify the browse_template which will be used to display browsing items.
4.Another important parameter is number_of_levels which specifies the maximum number of browsing levels visitors will see. When this level is reached visitors will see browsing results (i.e. listings). If you leave this setting blank, browsing results will be displayed when all browsing levels (from Browsing Page and category Browsing Settings) are exhausted.

 

The layout of the browsing search results page is configured in the settings for each category in the parameter "Search Results Template" (by default iLister uses the template called “default_search_result_item.tpl” located at /modules/classifieds/apps/FrontEnd/templates/$ThemeName/category_templates/display.

 

 

Site Visitor Experience

Visitor opens your iLister-based website and see the homepage with browsing block (or several blocks).

 

Usually there is a section "Browse By State", "Browse By Category" or "Browse By..." specific field (like MakeModel).
